Using .windsurfrules

Our .windsurfrules file now defines two rule sets to guide HTML-post handling:

1. html_fullscreen_posts

  • Triggered by full_screen: true in front matter
  • Allows full-document tags (<!DOCTYPE>, <html>, <head>, <body>)
  • Enables global body styles and a full-viewport background
  • Overrides Hugo’s layout (preserve_hugo_layout: false)
  • Ideal for immersive, background-like demos

2. html_blog_posts

  • Default when full_screen: false or omitted
  • Forbids top-level tags to preserve theme structure
  • Requires scoped CSS inside a container (.threejs-container)
  • Keeps Hugo’s header/footer and responsive design
  • Best for small interactive embeds within articles

How to write custom HTML posts

  • Set full_screen in your front matter (true or false).
  • Write your HTML according to the matched rule:
    • Full-screen: include top-level tags and global styles.
    • Inline: wrap content in a <div class="threejs-container"> and use scoped selectors.
  • Place <script type="module"> blocks at the bottom of your content.
